- **Why growing:** Минимум 1 публичный кейс (@astroboysoup) с реальными результатами, 8 упоминаний claude/claude-code в Twitter за один день — паттерн замены SaaS на internal tools становится mainstream

- **Risk:** Claude Code может генерировать нерабочий код для сложных интеграций; инструмент требует технических знаний для деплоя

### Claude Code в tmux + стиринг (cctool)

- **Scores:** novelty=8, traction=6, utility=7, testability=6, business_impact=7, noise_risk=3
- **Final score:** 7.0
- **Confidence:** medium
- **Test steps:**
  1. Найти репозиторий @slobkebap (cc-tmux или аналогичное название) через его Twitter-профиль

- **Risk:** Репозиторий может быть в раннем/сыром состоянии; инструмент требует настройки tmux и Unix-окружения

### agent-tracer-2 (OpenTelemetry observability для AI-агентов)

- **Category:** ai_automation
- **What:** OpenTelemetry-нативный, local-first инструмент наблюдаемости для AI-агентов с хранением в DuckDB. Позволяет трассировать, воспроизводить и отлаживать поведение агентов
- **Problem:** AI-агенты непрозрачны: непонятно, почему агент принял то или иное решение, какой инструмент вызвал какое поведение. Стандартные APM-инструменты не подходят для LLM-агентов
- **Why growing:** Новый репозиторий с фокусом на local-first и OpenTelemetry-стандарт — ответ на реальную боль разработчиков агентов. Ниша observability для LLM быстро развивается
- **Evidence:** GitHub: bettyguo/agent-tracer-2, ⭐7, is_new=true, OpenTelemetry-native + DuckDB описание
- **Scores:** novelty=8, traction=3, utility=8, testability=7, business_impact=7, noise_risk=2
- **Final score:** 6.6
- **Confidence:** medium
- **Test steps:**
  1. Клонировать репозиторий https://github.com/bettyguo/agent-tracer-2 и изучить README
  2. Подключить к существующему AI-агенту (LangChain, CrewAI или самописный) через OpenTelemetry SDK
  3. Запустить 5–10 агентных задач, изучить трассы в DuckDB/интерфейсе и оценить читаемость логов
- **Metric:** Полнота трассировки (% вызовов инструментов, пойманных трейсером); время на обнаружение причины ошибки агента
- **Expected:** Видимость в реальном времени вызовов инструментов, токенов, цепочек решений агента без внешних зависимостей
- **Risk:** 7 звёзд — очень ранний проект, документация может быть неполной; интеграция может потребовать кастомного кода
